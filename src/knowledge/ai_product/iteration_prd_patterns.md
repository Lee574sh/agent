# AI 产品迭代 PRD 知识包

## 1. 核心架构与模块规范
在进行业务下钻时，标准的版本迭代往往必须审查/包含以下通用控制模块，不能遗漏在业务模块之外：
- [数据源接入与预处理]
- [大模型推理调度引擎]
- [系统离线/弱网模式策略]
- [前端历史会话搜索及缓存]
- [知识库入库/索引更新机制]

## 2. NFR (非功能要求) Checklist (Acceptance Worker)
- **性能与限流**：需标明搜索响应上限（如 <2s），对话长连接的并发限制。
- **降级与容灾**：当大模型 API 5XX/429 时，UI该渲染何种失败态并提供重试机制。
- **数据安全**：是否有敏感词脱敏、用户隐私字段截断？
- **日志与可观测**：是否全链路埋点？用户踩/赞等标注数据的留存机制。

## 3. 测试与验收关注点
1. 对话记忆 Token 截断边缘值（爆窗时是否妥善处理）。
2. 断网后点击发送并重连后的队列重试动作。
3. 搜索的高亮对齐，命中标题 vs 命中知识库段落的先后权重。
